# Butler Neo-Embedded 系统实用操作流程

本指南介绍了如何使用 C 语言前端控制器、Python 大脑与 STM32 硬件节点进行协同工作。

## 1. 启动阶段 (Initialization)
1. **连接硬件**: 将 STM32 通过 USB-TTL 模块连接至 PC。
2. **启动控制器**:
   ```bash
   cd programs/bcli
   ./butler-controller /dev/ttyUSB0
   ```
3. **状态确认**: 观察界面输出的 `[ SYSTEM STATUS ]`，确保 Brain 和 STM32 均显示为 `Active/Connected`。

## 2. 全自动克隆模式 (Recommended)
这是最快的使用方式，适用于快速复制卡片：
1. 输入命令: `nfc auto`
2. **第一步**: 放置**源卡**。系统会自动识别、读取并将其镜像保存至外部 Flash 芯片。
3. **第二步**: 看到提示后移开源卡。
4. **第三步**: 放置**目标卡**。系统会自动检测到新卡并将镜像烧录进去。
5. **结束**: 界面提示“全自动克隆完成”后即可拿走卡片。

## 3. 手动管理模式
适用于需要对特定卡槽进行操作的场景：
1. **识别**: 输入 `nfc scan` 查看 UID。
2. **保存**: 输入 `nfc clone` 将当前卡片数据保存到存储芯片。
3. **查看**: 输入 `storage info` (待扩展) 查看已保存的卡片信息。
4. **烧录**: 输入 `nfc burn <slot>` (例如 `nfc burn 0`) 将指定位置的数据写入新卡。

## 4. 智能交互模式
利用 Python 大脑进行复杂操作：
- 输入 `ask <问题>`: 例如 `ask "这张卡的数据格式是什么？"`。C 语言前端会调用 Python 后端进行分析并返回结果。

## 5. 常见问题
- **STM32 无响应**: 检查串口号是否正确，以及 `main.c` 中的波特率是否匹配 (默认 115200)。
- **存储失败**: 确保外部存储芯片已正确连接至 STM32 的 SPI 接口，并已在 `butler_storage.c` 中注册。
