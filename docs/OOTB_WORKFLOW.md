# Butler "开箱即用" 工作流程说明

为了提供零配置的上手体验，Butler 实现了全自动的初始化流程。以下是从下载到正式使用的完整过程：

## 1. 获取项目
用户通过 `git clone` 或下载压缩包获取 Butler 源代码。

## 2. 一键启动
用户只需根据操作系统双击对应的启动脚本：
- **Windows**: `run.bat` 或 `run_modern.bat`
- **Linux/macOS**: `run.sh` 或 `run_modern.sh`

## 3. 自动化后台处理 (静默执行)
脚本启动后，会立即在后台自动完成以下任务，无需用户干预：
- **配置文件初始化**: 自动检测 `.env` 是否存在。如果缺失，将根据 `.env.example` 自动创建一个初始版本。
- **依赖环境检测**: 自动检测 Python 环境和第三方库。
- **自动安装依赖**: 如果检测到缺少核心运行库，系统将自动调用 `dependency_manager` 以**便携模式 (Portable Mode)** 将所有依赖安装到项目目录下的 `lib_external` 文件夹。这确保了用户主系统的纯净，同时实现了“解压即用”。

## 4. 智能配置引导 (交互执行)
当系统核心逻辑启动后，会检测关键 API 密钥（如 DeepSeek）：
- **如果密钥缺失**: 系统会根据当前启动模式（经典或现代）弹出美观的 **配置向导 (Config Wizard)** 界面。
  - **经典模式**: 弹出基于 Tkinter 的原生对话框。
  - **现代模式**: 自动跳转到基于 Web 的玻璃拟态配置页面。
- **如果密钥已存在**: 直接进入主功能界面。

## 5. 核心功能就绪
配置完成后，Butler 将正式启动：
- 语音服务、NLU 引擎、记忆系统等自动初始化。
- 用户可以立即开始对话或执行任务。

---
**提示**: 对于高级用户，所有的自动配置依然可以通过手动编辑 `.env` 或 `config/system_config.yaml` 来覆盖。
