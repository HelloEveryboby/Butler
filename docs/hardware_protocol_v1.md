# Butler 系统：跨平台硬件通信协议 (HCP) v1.0

## 1. 系统架构
*   **感知/执行层 (STM32):** 直接操作物理设备 (LED, 电机, 传感器)。
*   **桥接层 (Butler-Runner Go):** 串口转发，充当高性能代理。
*   **决策层 (Butler-Main Python):** 业务逻辑控制 (如加密失败后的报警锁定)。

## 2. 通信协议格式 (Hex Protocol)
数据包为固定 10 字节。

| 偏移 | 字段名 | 长度 | 描述 |
| :--- | :--- | :--- | :--- |
| 0x00 | Header | 1 Byte | 固定 `0xAA` |
| 0x01 | Type | 1 Byte | `0x01`: 控制, `0x02`: 查询, `0x03`: 报警 |
| 0x02 | Device | 1 Byte | `0x10`: LED, `0x20`: 电机, `0x30`: 传感器, `0x40`: NFC, `0xFF`: 系统 |
| 0x03 | Action | 1 Byte | `0x00`: 关闭/锁定, `0x01`: 开启/解锁, `0x02`: 闪烁 |
| 0x04-0x07 | Data | 4 Bytes | 4 字节数据 (Big-Endian), 如 PWM、RGB、6位核心码校验等 |
| 0x08 | CheckSum | 1 Byte | 前 8 位异或校验 |
| 0x09 | End | 1 Byte | 固定 `0x55` |

## 3. 指令集示例 (Hex)
*   **开启绿灯:** `AA 01 10 01 00 00 FF 00 4B 55`
*   **系统自毁/紧急锁定:** `AA 03 FF 00 DE AD BE EF XX 55`
*   **NFC 状态查询:** `AA 02 40 00 00 00 00 00 E8 55`

## 4. 实施清单

### A. Python 端实现 (`package/device/hardware_manager.py`)
使用 `pyserial` 进行串口读写，支持回调机制。

### B. STM32 模板 (`assets/stm32_template/cmd_parser.c`)
定义串口中断处理函数，校验 Header、End 和 CheckSum，然后根据 Type 执行相应外设操作。

### C. Go 代理逻辑 (Butler-Runner)
通过 WebSocket 接收 Python 的 JSON 指令，转发为 Hex 包，并将硬件反馈实时推送到 Python 端。
